**Key implementation steps:**

1. **CDN loader config (at module level, outside component):**
```typescript
import { loader } from "@monaco-editor/react";
loader.config({ paths: { vs: "https://cdn.jsdelivr.net/npm/monaco-editor@0.55.1/min/vs" } });
```

2. **Dynamic import for Monaco Editor (prevents SSR crash per D-01):**
```typescript
import dynamic from "next/dynamic";
const MonacoEditor = dynamic(() => import("@monaco-editor/react").then(m => ({ default: m.default })), { ssr: false });
```
Note: The component file itself can be "use client" — the dynamic import is needed because Monaco accesses browser globals during initialization.

3. **Tab state (per D-03):**
```typescript
const [tabs, setTabs] = useState<EditorTab[]>([]);
const [activeTabPath, setActiveTabPath] = useState<string | null>(null);
```

4. **React to selectedFilePath prop changes (per D-04):**
Use a useEffect on `[selectedFilePath, worktreePath]`:
- If selectedFilePath is null: do nothing
- If a tab with that path already exists: setActiveTabPath(selectedFilePath)
- Otherwise: call readFileContent(worktreePath, relativePath) where relativePath is derived by stripping worktreePath prefix from selectedFilePath. Then add new tab with content, set as active.
- relativePath derivation (browser-safe, no path module): `selectedFilePath.startsWith(worktreePath + "/") ? selectedFilePath.slice(worktreePath.length + 1) : selectedFilePath`
- filename derivation: `relativePath.split("/").pop() ?? relativePath`

5. **Monaco onMount (per D-06, saves with Ctrl+S):**
```typescript
const handleEditorMount = (editor: any, monaco: any) => {
  editorRef.current = editor;
  editor.addAction({
    id: "save-file",
    label: "Save File",
    keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyCode.KeyS],
    run: async () => {
      const tab = tabs.find(t => t.path === activeTabPath);
      if (!tab || !tab.isDirty) return;
      try {
        await writeFileContent(worktreePath, tab.relativePath, tab.content);
        setTabs(prev => prev.map(t => t.path === tab.path ? { ...t, isDirty: false } : t));
        showToast("success");
      } catch {
        showToast("error");
      }
    },
  });
};
```
IMPORTANT: The keybinding action closure captures stale state. Use a ref to hold the current active tab:
```typescript
const activeTabRef = useRef<EditorTab | null>(null);
// Keep ref in sync:
useEffect(() => {
  activeTabRef.current = tabs.find(t => t.path === activeTabPath) ?? null;
}, [tabs, activeTabPath]);
```
Then in the save action, read `activeTabRef.current` instead of closure state.

6. **Dirty tracking (per D-08):**
Use Monaco Editor's `onChange` prop (fires with new value string):
```typescript
onChange={(value) => {
  if (value === undefined) return;
  setTabs(prev => prev.map(t =>
    t.path === activeTabPath
      ? { ...t, content: value, isDirty: true }
      : t
  ));
}}
```

7. **Theme sync (per D-05):**
```typescript
const { resolvedTheme } = useTheme();
const monacoTheme = resolvedTheme === "dark" ? "vs-dark" : "light";
// Pass as theme prop to MonacoEditor
// Also run in useEffect:
useEffect(() => {
  import("@monaco-editor/react").then(({ loader: l }) => {
    l.init().then(monaco => monaco.editor.setTheme(monacoTheme));
  });
}, [monacoTheme]);
```

8. **Toast (per D-07):**
Simple local state toast (no library):
```typescript
const [toast, setToast] = useState<{ type: "success"|"error"; visible: boolean }>({ type: "success", visible: false });
function showToast(type: "success"|"error") {
  setToast({ type, visible: true });
  setTimeout(() => setToast(prev => ({ ...prev, visible: false })), 3000);
}
```
Render: `<div className="absolute bottom-4 right-4 z-50 rounded-md px-4 py-2 text-sm shadow-md ...">` — green background on success, red on error.

9. **Tab switching:** When activeTabPath changes, call `editor.setModel(monaco.editor.getModel(monaco.Uri.parse("file://" + path)))`. Store models in a `useRef<Map<string, any>>(new Map())`. Create model on first open, retrieve on tab switch.
