## Spacing Scale

Declared values (multiples of 4, matching Tailwind v4 defaults):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (p-1) |
| sm | 8px | Tab close button padding, status dot gap (p-2) |
| md | 16px | Panel padding, tab horizontal padding (p-4) |
| lg | 24px | Section gaps (p-6) |
| xl | 32px | Layout-level gaps (p-8) |
| 2xl | 48px | Major panel breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Editor tab height: 36px (py-2 + text-sm line-height) — not a clean multiple of 4 in isolation but resolves to 36px via Tailwind
- Dirty dot indicator: 6px diameter (w-1.5 h-1.5) — visual cue only, no layout impact
- Tab bar scrollbar: 6px wide — matches existing custom scrollbar in globals.css

Source: `src/app/globals.css` scrollbar width + existing component patterns

## Color

| Role | CSS Variable | Light Value | Dark Value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | oklch(0.97 0.005 260) | oklch(0.15 0.005 260) | Editor surface, panel background |
| Secondary (30%) | `--card` / `--sidebar` | oklch(0.94 0.005 260) | oklch(0.19 0.005 260) | Tab bar background, inactive tabs |
| Accent (10%) | `--primary` | oklch(0.65 0.14 75) | oklch(0.80 0.14 75) | Active tab underline indicator, dirty dot color |
| Destructive | `--destructive` | oklch(0.55 0.2 25) | oklch(0.65 0.2 25) | Save error toast border, error state text |
| Muted text | `--muted-foreground` | oklch(0.50 0.008 260) | oklch(0.65 0.008 260) | Inactive tab filenames, line number gutter |
| Border | `--border` | oklch(0.82 0.008 260) | oklch(0.28 0.008 260) | Tab bar bottom border, editor container border |

Accent reserved for:
1. Active tab bottom underline (2px solid `--primary`)
2. Dirty dot indicator on unsaved tabs (`●` character in `--primary` color)
3. Focus ring on the editor container (inherited from `outline-ring/50` global)

Do NOT use accent on: tab background fill, hover states, general interactive elements.

Monaco theme mapping (ED-05):
- `resolvedTheme === "dark"` → Monaco theme `"vs-dark"` (dark background #1e1e1e, close enough to `--background` dark)
- `resolvedTheme === "light"` → Monaco theme `"light"` (white background, close to `--background` light)

Source: `src/app/globals.css` color tokens, CONTEXT.md D-05, existing `STATUS_COLORS` patterns in `task-page-client.tsx`

## Interaction Contracts

### ED-01: File Open
- Trigger: User clicks a file node in FileTree (`onFileSelect` callback with absolute path)
- Behavior: If tab exists for that path, activate it. If new path, fetch content via `readFile` server action, create Monaco model, add tab, activate it.
- Loading state: Monaco `loading` prop shows a spinner matching `--muted` background; no content flash.
- Empty editor (no file selected): Show centered placeholder text "Select a file to open" in `text-sm text-muted-foreground`.

### ED-02: Save (Ctrl+S / Cmd+S)
- Trigger: `editor.addAction()` with `keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyCode.KeyS]`
- Behavior: Call `writeFile` server action with worktreePath + relativePath + content. On success: clear dirty flag, show success toast. On failure: show error toast.
- Browser save dialog: Must be prevented (`preventDefault()` on the action).

### ED-03: Dirty Indicator
- Trigger: Monaco `onDidChangeModelContent` event
- Behavior: Set `isDirty: true` on the corresponding tab. Display `●` (U+25CF) prepended to filename in tab: `● filename.ts`.
- Clean state: Plain `filename.ts` (no prefix).
- Dirty dot color: `text-primary` (amber accent).

### ED-04: Multi-Tab Switching
- Tab bar: Horizontal scrollable row of tabs above the editor. Max display width fills available panel width.
- Switching: Clicking a tab sets it as active. Monaco editor swaps to the model for that path. No re-fetch from server.
- Close tab: X button (8px × 8px, `text-muted-foreground hover:text-foreground`) on each tab. Closing a dirty tab: no confirmation — discard silently (out-of-scope complexity per CONTEXT.md).
- Maximum open tabs: No hard limit. Tabs scroll horizontally via ScrollArea.

### ED-05: Theme Sync
- Trigger: `useTheme()` `resolvedTheme` change
- Behavior: Call `monaco.editor.setTheme("vs-dark" | "light")` inside a `useEffect` with `[resolvedTheme]` dependency.

## Layout Contract

### Files Tab Layout (inside right panel)

```
┌─────────────────────────────────────────┐
│  [tab: ● App.tsx ×] [tab: utils.ts ×]  │  ← EditorTabs, h-9 (36px), bg-card, border-b
├─────────────────────────────────────────┤
│                                         │
│           Monaco Editor                 │  ← flex-1, overflow-hidden
│         (fills remaining height)        │
│                                         │
│  [toast: 保存成功]  (bottom-right)       │  ← absolute, bottom-4 right-4, z-50
└─────────────────────────────────────────┘
```

- The Files tab content area is split: FileTree (left, fixed width ~240px) | Editor area (right, flex-1) using a nested flex row or additional Panel from react-resizable-panels.
- FileTree left pane width: 240px default, resizable if adding a handle. For Phase 21 simplicity: fixed 240px, no additional resize handle unless executor chooses otherwise.
- Editor area: `flex flex-col h-full` — tab bar on top, Monaco fills remaining height.
- Monaco container: `flex-1 min-h-0` to prevent flex child overflow. Must use `height: 100%` on the Monaco Editor component.

Source: CONTEXT.md code_context section, `task-page-client.tsx` existing panel layout
