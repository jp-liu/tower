Create `src/actions/assistant-actions.ts` with `"use server"` directive. Define:

**Constant:** `const ASSISTANT_SESSION_KEY = "__assistant__";`

**Export `ASSISTANT_SESSION_KEY`** so ws-server.ts and the API route can import it.

**Function `startAssistantSession(): Promise<void>`:**
1. Call `destroySession(ASSISTANT_SESSION_KEY)` to ensure fresh session (UX-01)
2. Read default CliProfile: `await db.cliProfile.findFirst({ where: { isDefault: true } })`
3. If no profile found, throw `new Error("No default CLI profile — configure one in Settings")`
4. Parse `profile.baseArgs` with `JSON.parse(profile.baseArgs) as string[]` inside try/catch
5. Parse `profile.envVars` with `JSON.parse(profile.envVars) as Record<string, string>` inside try/catch
6. Read system prompt: `await readConfigValue<string>("assistant.systemPrompt", "You are Tower Assistant, an AI operator for the Tower task management platform. You help users create, organize, query, and track tasks and projects using Tower MCP tools. You do NOT write or edit code — you are an operator, not a developer. Always respond in the same language the user uses.")`
7. Set `cwd = process.cwd()` (Tower project root)
8. Build `claudeArgs: string[]`:
   - Spread `profileBaseArgs`
   - Push `"--allowedTools"`, `"mcp__tower__*"` as two separate elements (BE-03)
   - Push `"--append-system-prompt"`, `systemPrompt` as two separate elements (BE-02)
9. Build `envOverrides: Record<string, string>` from `profileEnvVars` only — do NOT inject `AI_MANAGER_TASK_ID` (assistant has no task)
10. Call `createSession(ASSISTANT_SESSION_KEY, profileCommand, claudeArgs, cwd, () => {}, () => {}, envOverrides)` (BE-01)

**Function `stopAssistantSession(): Promise<void>`:**
1. Call `destroySession(ASSISTANT_SESSION_KEY)` (BE-05)

**Function `getAssistantSessionStatus(): Promise<"running" | "idle">`:**
1. Call `getSession(ASSISTANT_SESSION_KEY)`
2. If no session or `session.killed` is true, return `"idle"`
3. Otherwise return `"running"`

**Imports needed:**
- `createSession`, `destroySession`, `getSession` from `"@/lib/pty/session-store"`
