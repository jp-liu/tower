      baseBranch: string

    Implementation details (per D-06, reuse logic per Claude's discretion):
    1. Compute `worktreePath = path.join(localPath, ".worktrees", "task-" + taskId)`
    2. Compute `worktreeBranch = "task/" + taskId`
    3. `await mkdir(path.join(localPath, ".worktrees"), { recursive: true })`
    4. Call `execSync("git worktree list --porcelain", { cwd: localPath, encoding: "utf-8", timeout: 10000 })`
    5. If output includes `worktree <worktreePath>` (exact line match), return early with `{ worktreePath, worktreeBranch }` (reuse)
    6. Check if branch exists: `execSync("git branch --list task/" + taskId, { cwd: localPath, encoding: "utf-8", timeout: 5000 }).trim()`
    7. If branch exists (non-empty output): run `git worktree add "${worktreePath}" ${worktreeBranch}` (attach existing branch, no -b flag) — this preserves previous work per Open Question 1 in RESEARCH.md
    8. If branch does not exist: run `git worktree add -b ${worktreeBranch} "${worktreePath}" ${baseBranch}` (create new branch)
    9. Both git commands use `{ cwd: localPath, encoding: "utf-8", timeout: 30000 }` (30s timeout for potentially slow disks)
    10. On any execSync error, let the exception propagate with the original git error message

    let cwd = task.project!.localPath!;

    if (task.project!.type === "GIT" && task.baseBranch) {
      try {
        const { worktreePath, worktreeBranch } = await createWorktree(

        await db.taskExecution.update({

          data: { worktreePath, worktreeBranch },

        cwd = worktreePath;
        sendEvent({ type: "status", content: `Worktree ready: ${worktreeBranch}` });
