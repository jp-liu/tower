## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` — `oklch(0.97 0.005 260)` light / `oklch(0.15 0.005 260)` dark | Preview iframe container, panel background |
| Secondary (30%) | `var(--card)` — `oklch(0.94 0.005 260)` light / `oklch(0.19 0.005 260)` dark | Preview toolbar row, settings form rows |
| Accent (10%) | `var(--primary)` — `oklch(0.65 0.14 75)` light / `oklch(0.80 0.14 75)` dark | Run button (active state), address bar focus ring |
| Destructive | `var(--destructive)` — `oklch(0.55 0.2 25)` light / `oklch(0.65 0.2 25)` dark | Stop button (when server running), error state text |

Accent reserved for:
- "Run" button background when dev server is stopped (ready-to-start state)
- Address bar `ring` focus outline
- Active tab underline border (reuses existing `data-[state=active]:border-primary` pattern)

Status indicator colors (semantic, not accent):
- Running: `text-emerald-400 bg-emerald-500/10 border border-emerald-500/20` — mirrors existing IN_PROGRESS color pattern
- Stopped: `text-muted-foreground bg-muted` — neutral
- Error: `text-red-400 bg-red-500/10 border border-red-500/20` — distinct from destructive-button red

Source: `globals.css` CSS variables + existing `STATUS_COLORS` pattern in `task-page-client.tsx`.

## Interaction Contract

### Preview Panel Layout (PV-02)

```
┌────────────────────────────────────────────────────────┐
│ Toolbar row (h-11, px-3, bg-card, border-b)            │
│  [●Running / ○Stopped / ✕Error badge]  [Run]/[Stop]    │
│  [Refresh icon btn]  [Open Terminal icon btn]           │
├────────────────────────────────────────────────────────┤
│ Address bar row (h-9, px-3, py-1, bg-background)       │
│  [http://localhost:3000______________] ← input, w-full  │
├────────────────────────────────────────────────────────┤
│                                                        │
│           <iframe> fills remaining height              │
│           src={previewUrl}, key={iframeKey}            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### States

| State | Visual |
|-------|--------|
| No worktree | Eye icon centered + "启动任务后可使用预览面板" body text (mirrors `editor.noWorktree` pattern) |
| Backend project | Preview tab hidden from tab bar — existing condition `task.project?.type !== "BACKEND"` in task-page-client.tsx |
| Server stopped | Run button uses `variant="default"` (primary accent fill); status badge shows "已停止" in muted |
| Server starting | Run button disabled + `Loader2` spinner icon; status badge shows "启动中..." in muted |
| Server running | Run button replaced by Stop button (`variant="destructive"`); status badge shows "运行中" in emerald |
| Server error | Stop button remains; status badge shows "启动失败" in red; error detail in small text below toolbar |
| iframe navigating | No skeleton — iframe `loading` attribute handles browser-native loading state |

### Segmented Control: Project Type (PV-01)

Reuse exact button group pattern from `general-config.tsx` (`inline-flex rounded-md border bg-muted p-1 gap-1`). Two options:
- "前端 / Frontend" (default, value: `FRONTEND`)
- "后端 / Backend" (value: `BACKEND`)

Active option: `bg-background text-foreground shadow-sm`
Inactive option: `text-muted-foreground hover:text-foreground`

Placement in project dialogs: below project name field, above Git URL field.

### Auto-Refresh (PV-06)

`CodeEditor` calls `onSave?.()` after `writeFileContent` succeeds. `PreviewPanel` increments `iframeKey` state on `onSave` invocation. No debounce — save is already user-initiated (Ctrl+S). No WebSocket.

### Open in Terminal (PV-04)

Button: `variant="ghost" size="icon"` with `Terminal` lucide icon. Calls server action `openInTerminal(worktreePath)`. No confirmation. On error: toast with "无法打开终端，请检查终端配置".

## iframe Security Contract

The preview iframe renders localhost URLs only. Apply these sandbox attributes:

```html
<iframe
  src={previewUrl}
  key={iframeKey}
  className="h-full w-full border-0"
  allow="clipboard-read; clipboard-write"
  referrerPolicy="no-referrer-when-downgrade"
/>
```

Do NOT apply `sandbox` attribute — it blocks JavaScript execution and breaks all dev servers.
Do NOT set `X-Frame-Options` on the Next.js dev server — it is not the host being embedded.
No `postMessage` bridge needed (auto-refresh via iframe key increment is sufficient).

Source: CONTEXT.md `## Claude's Discretion` — iframe sandbox attributes at researcher's discretion.

## Process Security Contract

Preview command execution follows existing project security constraint:

- Command stored as `previewCommand` string (e.g. `"pnpm dev"`).
- Split by whitespace into args array at invocation time: `["pnpm", "dev"]`.
- Passed to `child_process.spawn(args[0], args.slice(1), { cwd: worktreePath, shell: false })`.
- Never passed to shell string interpolation.
- `shell: false` is mandatory (security constraint from STATE.md blockers).

"Open in Terminal" uses `execFileSync("open", ["-a", terminalApp, worktreePath])` — args array, no shell.

Source: STATE.md Blockers — "previewCommand is split by whitespace into args array and passed to spawn(shell: false)".
