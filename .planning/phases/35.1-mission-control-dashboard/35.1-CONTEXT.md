# Phase 35.1: Mission Control Dashboard - Context

**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can view and interact with all running task terminals in a single `/missions` page across all workspaces. The page displays a configurable grid of terminal cards, each connected to a live PTY session. Users can launch new task executions, stop running ones, and navigate to the full task detail page.

</domain>

<decisions>
## Implementation Decisions

### Card Layout & Grid
- **D-01:** Grid uses preset dropdown (1x1, 2x1, 2x2, 3x2, 4x2, 3x3). User picks from fixed presets; choice persists in localStorage.
- **D-02:** Each card shows full breadcrumb header: `Workspace › Project › Task title` + running time + status badge + drag handle (≡) + open full view (↗) + stop button (×).
- **D-03:** Cards are reorderable via drag-and-drop using dnd-kit (already in project for Kanban board).
- **D-04:** Top toolbar bar: page title left, layout preset dropdown + "Launch Task" button right. Follows Kanban board header pattern.
- **D-05:** Empty state: centered icon + "No running tasks" message + "Launch Task" CTA button.

### Terminal Embedding
- **D-06:** Terminals auto-fit to card size using FitAddon (already used in TaskTerminal). When grid layout changes, terminals re-fit.
- **D-07:** Terminals are fully interactive — users can type input directly into any card. Each card connects its own WebSocket to the PTY session.

### Task Launcher Flow
- **D-08:** Launch dialog uses cascading selects: Workspace dropdown → Project dropdown → Task list. Filters to TODO/IN_PROGRESS tasks.
- **D-09:** Clicking "Launch" starts execution immediately (calls startPtyExecution), creates a new card with live terminal.

### Lifecycle & Auto-remove
- **D-10:** Page polls server every 3-5 seconds to detect execution status changes (no additional WebSocket channel needed).
- **D-11:** When execution completes or is stopped, card shows "Completed"/"Stopped" status for ~3 seconds, then fades out and is removed from the grid.

### Claude's Discretion
- Terminal theme matching (dark/light) — follow existing TaskTerminal pattern
- Exact polling interval within the 3-5 second range
- CSS grid implementation details
- Animation/transition specifics for card fade-out
- Card minimum size constraints for readability

</decisions>
