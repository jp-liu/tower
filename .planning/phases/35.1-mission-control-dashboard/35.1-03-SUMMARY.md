## What Was Built

**Type Fixes:**

The Base UI Select component's `onValueChange` prop has the signature:
```ts
(value: string | null, eventDetails: SelectRootChangeEventDetails) => void
```

Four handlers across two files needed updates to accept `null`:

- `missions-client.tsx`: `handlePresetChange` changed from `(newId: string)` to `(newId: string | null)` with `if (!newId) return` guard
- `task-picker-dialog.tsx` (3 handlers):
  - Workspace `onValueChange`: inline lambda updated to `(v) => { setSelectedWsId(v ?? ""); ... }`
  - Project `onValueChange`: changed from direct `setSelectedProjectId` to `(v) => setSelectedProjectId(v ?? "")`
  - Task `onValueChange`: changed from direct `setSelectedTaskId` to `(v) => setSelectedTaskId(v ?? "")`
