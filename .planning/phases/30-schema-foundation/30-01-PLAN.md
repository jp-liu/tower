Add the following to prisma/schema.prisma:

1. Add a new `CliProfile` model AFTER the `AgentConfig` model block:

```prisma
model CliProfile {
  id          String   @id @default(cuid())
  name        String   @unique
  command     String
  baseArgs    String   @default("[]")
  envVars     String   @default("{}")
  isDefault   Boolean  @default(false)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

Field semantics (per CLIP-01 + STATE.md decision on `baseArgs`):
- `name`: unique profile identifier (e.g. "default", "custom-model")
- `command`: CLI executable (e.g. "claude")
- `baseArgs`: JSON-serialized string array of CLI flags (e.g. `["--dangerously-skip-permissions"]`). Use String not String[] because SQLite has no native array type. Consumers parse with JSON.parse().
- `envVars`: JSON-serialized key-value object for environment overrides (e.g. `{"ANTHROPIC_MODEL":"claude-sonnet-4-6"}`). Same SQLite string storage pattern as baseArgs.
- `isDefault`: marks the active profile; downstream code reads `where: { isDefault: true }`.

2. Add `callbackUrl` field to the existing `TaskExecution` model. Insert it AFTER the `exitCode` field and BEFORE `terminalLog`:

```prisma
  callbackUrl    String?
```

This is a nullable string for DATA-01 — stores the URL to notify when execution completes.

3. Run `npx prisma db push` to apply schema changes to the SQLite database.
4. Run `npx prisma generate` to regenerate the Prisma client with new types.
