  <action>
Create `scripts/post-tool-hook.js` — a Node.js script that runs as a Claude Code PostToolUse hook.

The script:
1. Reads JSON from stdin (Claude Code pipes tool use info). Set a 10s stdin timeout like existing hooks.
2. Checks `process.env.TOWER_TASK_ID` — if not set, exit 0 immediately (no-op outside Tower).
3. Checks `process.env.TOWER_API_URL` — if not set, exit 0.
4. Parses the stdin JSON. The structure is: `{ tool_name, tool_input, session_id, cwd, ... }`
5. Only processes tools that write files: `Write`, `Edit`, `MultiEdit`. For these tools, extract the file path from `tool_input.file_path` (Write/Edit/MultiEdit all use `file_path`).
6. If no file_path found, exit 0.
7. Check if the file extension matches the allowed types. The hook fetches the whitelist from `TOWER_API_URL + "/api/internal/hooks/config"` (we will create this in the upload route file as a GET handler). If fetch fails, use hardcoded default: `["png","jpg","jpeg","gif","webp","svg","pdf","md","txt","json"]`.
8. Check if file exists on disk (it should since PostToolUse fires after the tool ran). If not, exit 0.
9. POST to `TOWER_API_URL + "/api/internal/hooks/upload"` with JSON body `{ taskId: TOWER_TASK_ID, filePath: absoluteFilePath }`.
10. Exit 0 regardless of upload success (hook must not block Claude Code).

Also add a GET handler to the upload route file (`src/app/api/internal/hooks/upload/route.ts`) that returns the configured type whitelist:
- Read `hooks.autoUploadTypes` from SystemConfig (via `readConfigValue`)
- Return JSON `{ types: [...] }`
- Guard with `requireLocalhost`

The hook script should:
- Use `#!/usr/bin/env node` shebang
- Use only Node.js built-in modules (no external deps) — `fs`, `path`, `http`/`https`
- Use `http.request` (not fetch) for compatibility with Node 16+
- Handle errors silently (never crash, always exit 0)
- Include a header comment explaining purpose
