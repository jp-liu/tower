**task-flow.spec.ts** (E2E-01):
Create a serial test suite "Task Lifecycle Flow":

1. Test "navigate to workspace": Go to `/workspaces`, click the first workspace in sidebar (or create one if none exist using the pattern from smoke.spec.ts). Store workspaceId from URL.

2. Test "create a new task": Click the create-task button (look for a Plus/Add button in the board area or header). Fill in the dialog: title = "E2E Test Task " + Date.now(), leave defaults. Submit. Wait for the dialog to close. Verify a card with that title appears in the TODO column.

3. Test "change task status to IN_PROGRESS": Find the task card by its title text. Right-click to open context menu (the app has context menus on task cards per Phase 27). Click the "In Progress" / "进行中" menu item. Verify the card moves to the IN_PROGRESS column (check the card is now under the IN_PROGRESS column header).

4. Test "mark task as DONE": Right-click the task card again. Click "Done" / "完成" menu item. Verify the card moves to the DONE column.

Use `page.context().storageState()` or test.describe.serial to share state across tests.

**settings-flow.spec.ts** (E2E-03):
Create a serial test suite "Settings Persistence Flow":

1. Test "navigate to settings": Go to `/settings`. Verify the settings page loads (nav with "General" visible).

2. Test "change theme to dark": Click the "Dark" / "深色" theme button in the segmented control. Verify `<html>` gets class "dark".

3. Test "verify theme persists after reload": Reload the page (`page.reload()`). Wait for settings to load. Verify `<html>` still has class "dark". Then switch back to "System" / "跟随系统" to clean up.

4. Test "change language and verify": Click the language selector, switch to "en" (English). Verify text changes to English (e.g. "General" label appears instead of Chinese). Reload. Verify language persists. Switch back to "zh" to clean up.

Important patterns:
- The app uses next-themes for theme switching — the class is on `<html>` element
- Language is stored in localStorage and applied via I18nProvider
- Use `page.locator("button").filter({ hasText: /Dark|深色/ })` pattern for bilingual matching
- Add reasonable `waitForTimeout(500)` after theme/locale changes for hydration
