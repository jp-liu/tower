**chat-flow.spec.ts** (E2E-02):
Create a serial test suite "Chat Assistant Flow".

First, read the assistant-provider.tsx to find how the assistant panel is opened. Look for:
- A keyboard shortcut (likely Cmd+K or similar)
- A button in the layout/sidebar that toggles the assistant
- The `useAssistant()` hook's `open` method

Read assistant-chat.tsx to find:
- The textarea/input element for typing messages
- The send button
- The message list container
- How user vs assistant bubbles are distinguished

**Test structure:**

1. Test "open assistant panel":
   - Navigate to any page (e.g. `/workspaces`)
   - Trigger the assistant panel open (keyboard shortcut or click the assistant button)
   - Verify the panel is visible (look for the assistant panel container, the chat input, or a panel header)

2. Test "send a text message":
   - Find the chat textarea/input inside the assistant panel
   - Type a simple message like "Hello, please respond with just the word OK"
   - Click send (or press Enter/Cmd+Enter depending on the implementation)
   - Verify the user message bubble appears containing the sent text
   - Use `page.locator` with text content match

3. Test "receive assistant response":
   - This test waits for an assistant response bubble to appear
   - The assistant panel uses a real Claude CLI PTY session, so this needs a generous timeout
   - Use `expect(assistantBubble).toBeVisible({ timeout: 120_000 })` or similar
   - If the test environment has no Claude CLI, the test should gracefully skip: check if a response OR an error message appears within timeout
   - IMPORTANT: This test may be flaky in CI without Claude CLI. Add a comment noting this dependency.

4. Test "paste image shows thumbnail":
   - Focus the chat input
   - Simulate a clipboard paste event with an image. Use Playwright's `page.evaluate` to dispatch a paste event with a File in clipboardData:
     ```typescript
     await page.evaluate(() => {
       const input = document.querySelector('textarea') || document.querySelector('[contenteditable]');
       if (!input) return;
       // Create a small 1x1 PNG as a blob
       const base64 = 'iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNk+M9QDwADhgGAWjR9awAAAABJRU5ErkJggg==';
       const binary = atob(base64);
       const bytes = new Uint8Array(binary.length);
       for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i);
       const file = new File([bytes], 'test-image.png', { type: 'image/png' });
       const dt = new DataTransfer();
       dt.items.add(file);
       const pasteEvent = new ClipboardEvent('paste', {
         clipboardData: dt,
         bubbles: true,
         cancelable: true,
       });
       input.dispatchEvent(pasteEvent);
     });
     ```
   - Wait for a thumbnail to appear (look for an img element in the thumbnail strip area, or a container with thumbnail class)
   - Verify at least one thumbnail image is visible
   - This tests the paste-to-upload confirmation path without needing to actually send the image to Claude

**Important considerations:**
- The assistant uses a communication mode setting (terminal vs chat). The test needs chat mode for message bubbles. Check if the test needs to set this via the settings page first, or if there's a way to set it programmatically.
- If the assistant defaults to terminal mode, add a setup step that navigates to settings and switches to chat mode first.
- Use `test.slow()` for the response test to extend the timeout.
- Add `test.skip` conditions for tests that require Claude CLI if the environment doesn't have it.
