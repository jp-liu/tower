```typescript
export type MessageRole = "user" | "assistant" | "thinking" | "tool";

export interface ChatMessage {
  id: string;
  role: MessageRole;
  content: string;
  toolName?: string;
  isStreaming?: boolean;
}

export interface UseAssistantChatReturn {
  messages: ChatMessage[];
  isThinking: boolean;
  sendMessage: (text: string) => void;
  wsStatus: "connecting" | "connected" | "disconnected";
}

export function stripAnsi(text: string): string { ... }
export function useAssistantChat(opts: { enabled: boolean; worktreePath: string | null }): UseAssistantChatReturn { ... }
```

Implementation details:
- **ANSI regex**: `const ANSI_REGEX = /\x1B(?:[@-Z\\-_]|\[[0-?]*[ -/]*[@-~]|\](?:[^\x07\x1B]|\x1B[\\])*(?:\x07|\x1B\\))/g;`
- **State machine** with states: `idle`, `user`, `assistant`, `thinking`, `tool`
- **Line buffer**: accumulate incoming chunks, split on `\n`, process only complete lines, keep last partial in buffer
- **Transition heuristics**:
  - Line starts with `>` or `❯` in idle/assistant state -> user segment
  - Line contains `╭` box-drawing -> potential new message boundary, finalize current segment
  - Line contains `╰` box-drawing -> end of current block
  - Line contains thinking unicode (hourglass, spinner patterns) -> thinking segment
  - Line contains `Tool:` or tool-call formatted output -> tool segment
  - Otherwise in assistant context -> accumulate to assistant content
- **WebSocket**: connect to `ws://localhost:${wsPort}/terminal?taskId=__assistant__` via `getConfigValue<number>("terminal.wsPort", 3001)`. Import `ASSISTANT_SESSION_KEY` from `@/lib/assistant-constants`.
- **sendMessage**: `socket.send(text + "\n")` if connected
- **isThinking**: derived from last message having `role === "thinking"` and `isStreaming === true`
- **Message IDs**: use incrementing counter prefix + timestamp: `msg-${counter++}-${Date.now()}`
- **Cleanup**: close WebSocket on unmount or when `enabled` becomes false

The hook does NOT import xterm.js — it is a pure data hook.
