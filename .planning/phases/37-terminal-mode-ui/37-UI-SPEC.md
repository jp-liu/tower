## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, divider margins |
| sm | 8px | Compact element spacing, icon-to-label gap |
| md | 16px | Default element spacing, panel body padding |
| lg | 24px | Section padding, title bar padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Top bar height: 48px (h-12) — matches existing layout
- Assistant icon touch target: 32px (h-8) — matches existing top-bar button sizing convention
- Title bar height: 44px — provides comfortable touch target for close button

Source: `top-bar.tsx` uses `h-12` (48px) header, buttons use `h-8` (32px) per `.claude/rules/ui.md`

## Layout & Interaction Contract

### Panel Dimensions

| Mode | Dimension | Value |
|------|-----------|-------|
| Sidebar | Width | 420px fixed |
| Sidebar | Height | 100vh minus top-bar (calc(100vh - 48px)) |
| Dialog | Width | 600px |
| Dialog | Height | 70vh (min 480px, max 800px) |

Sidebar layout is **push**, not overlay: when `AssistantPanel` is open in sidebar mode, the main content area shrinks by 420px. The `layout-client.tsx` flex row adds the panel as a sibling to the main content div.

Source: `37-CONTEXT.md` — "Push layout for sidebar (UX-02)"

### Title Bar

Height: 44px
Content (left to right):
1. `Bot` icon from lucide-react — 16px, `text-muted-foreground`
2. Text label "Tower Assistant" — 14px semibold, `text-foreground`
3. Spacer (flex-1)
4. WebSocket status dot — 8px circle, colors: connecting = `bg-amber-400`, connected = `bg-emerald-400`, disconnected = `bg-rose-400`
5. Close button — `Button` variant ghost, `h-8 w-8`, `XIcon` 16px

Separator: 1px `border-b border-border` below title bar, above terminal body.

### Assistant Icon in Top Bar

Position: In the right-actions row, between the search box and the language toggle (before the divider).
Element: `<button>` (not a Link) with `Bot` icon (16px), `h-8 w-8`, same hover class as other top-bar buttons: `rounded-lg p-2 text-muted-foreground transition-colors hover:bg-accent hover:text-foreground`.
Active state (panel open): add `bg-accent text-foreground` to indicate open state.
Keyboard shortcut: Cmd+L (macOS) / Ctrl+L (Windows/Linux), registered in `useEffect` at layout level.

Source: `37-CONTEXT.md` specifics, `top-bar.tsx` button patterns

### Open/Close Lifecycle

| Action | Result |
|--------|--------|
| Click assistant icon | Call `toggleAssistant()` — POST `/api/internal/assistant` if opening |
| Press Cmd+L / Ctrl+L | Call `toggleAssistant()` |
| Press Escape | Close panel, DELETE `/api/internal/assistant` |
| Click close button in title bar | Close panel, DELETE `/api/internal/assistant` |
| Panel already open + Cmd+L | Close panel, DELETE `/api/internal/assistant` |

Session lifecycle: POST starts the PTY, DELETE destroys it. No auto-open on page load.

Source: `37-CONTEXT.md` decisions

### Loading State

While the assistant session is being created (POST in flight), the terminal body shows an opacity-40 overlay with a centered `Loader2` spinner (20px, `animate-spin`). This follows the existing loading pattern from `.claude/rules/ui.md`:
```
<div className="relative opacity-40 pointer-events-none">
  <Loader2 className="absolute inset-0 m-auto size-5 animate-spin" />
</div>
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard trap in sidebar | Sheet/Dialog components handle focus trap via base-ui primitives |
| Close on Escape | Sheet/Dialog natively handle Escape; additionally register at keyboard event level |
| Assistant icon accessible name | `aria-label={t("assistant.iconLabel")}` |
| Screen reader panel role | Sheet renders as `role="dialog"` via base-ui `Dialog` primitive |
| Terminal keyboard access | xterm.js is keyboard-accessible by default; focus terminal on open |
