## Spacing Scale

Declared values (multiples of 4, matching 8-point grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in tree nodes; badge internal padding |
| sm | 8px | Tree node vertical padding; context menu item padding |
| md | 16px | Tab bar horizontal padding; section padding |
| lg | 24px | Empty state vertical gap between icon and text |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Tree node indent per depth level: 12px (`pl-3`) — intentionally sub-8 to keep deep trees readable
- File/folder icon size: 14px (`h-3.5 w-3.5`) — consistent with tab bar icons in Phase 19
- Git status badge width: 16px fixed (`w-4`) — character badge (M/A/D), needs exact sizing
- Touch target minimum for tree node rows: 28px height — compact IDE-style tree, not touch-primary UI

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px (`text-[13px]`) | 400 (regular) | 1.5 | File/folder names in tree nodes |
| Label | 11px (`text-[11px]`) | 500 (medium) | 1.4 | Git status badges (M/A/D); section headers |
| Small | 12px (`text-xs`) | 400 (regular) | 1.5 | Context menu items; empty state body copy |
| Heading | 14px (`text-sm`) | 600 (semibold) | 1.2 | Tab bar active trigger; inline rename input |

Note: 13px body is intentional for this phase — file trees follow IDE convention (VS Code uses 13px). This overrides the 14px minimum used elsewhere in the app. No new font sizes are introduced; all map to Tailwind utilities.

Font weight palette: regular (400) + semibold (600). Medium (500) used only for git badges — treated as a label modifier, not a third weight family.

## Interaction Contract

### Tree Node States

| State | Visual |
|-------|--------|
| Default | `text-foreground bg-transparent` |
| Hovered | `bg-accent text-accent-foreground` |
| Selected (open in editor) | `bg-secondary text-secondary-foreground font-medium` |
| Inline rename active | Input replaces label text; `ring-1 ring-primary` focus ring |
| Being created (new entry) | Ghost row at insertion point with empty input; `italic text-muted-foreground` placeholder |

### Expand/Collapse

- Folders: click anywhere on row (icon, chevron, or name) toggles expansion
- Chevron icon: `ChevronRight` (collapsed) → `ChevronDown` (expanded), 12px, `transition-transform duration-150`
- Lazy load: fetch children on first expand; cache in component state until tree refresh

### Context Menu

- Trigger: `onContextMenu` on any tree node row
- Position: `{ top: event.clientY, left: event.clientX }` rendered via `createPortal` to `document.body`
- Dismiss: click outside (`mousedown` listener on document), or Escape key
- Menu items (in order): New File | New Folder | `<Separator>` | Rename | `<Separator>` | Delete
- Delete item: `text-destructive hover:bg-destructive/10` to signal danger before confirm dialog

### Inline Rename / Create

- Rename: context menu "Rename" replaces filename span with `<input>` pre-filled with current name
- Create: context menu "New File" / "New Folder" inserts ghost row at target location with empty `<input>`
- Enter: confirms and calls server action; on success tree refreshes; on error shows inline error text below input (`text-[11px] text-destructive`)
- Escape: cancels without server call; ghost row or edit reverts instantly

### Auto-refresh (FT-04)

- `setInterval(2000)` active only when `executionStatus === 'RUNNING'`
- `clearInterval` on component unmount or when status leaves RUNNING
- Refresh calls `listDirectory` for expanded paths only — do not re-collapse expanded folders on refresh
- New/modified/deleted files animate in with no transition (instant diff — keep it simple)

### Empty States

| Condition | Visual |
|-----------|--------|
| No worktree path (task not yet executed) | `FolderTree` icon (32px, `text-muted-foreground`) + heading + body copy (see Copywriting) |
| Worktree exists but empty | `Folder` icon (32px) + "Empty directory" heading |
| Load error | `AlertCircle` icon (32px, `text-destructive`) + error heading + retry link |

## Layout Integration

Phase 20 replaces the placeholder in `task-page-client.tsx` `TabsContent value="files"`.

```
TabsContent[value="files"] (flex-1 overflow-hidden)
  └── FileTree (h-full flex flex-col)
        ├── TreeHeader (px-2 py-1 border-b border-border) — optional refresh indicator
        └── ScrollArea (flex-1)
              └── TreeRoot (recursive FileTreeNode list)
```

The `FileTree` component receives:
- `worktreePath: string` — from `TaskExecution.worktreePath`
- `baseBranch: string` — from `Task.baseBranch`, for git diff status
- `executionStatus: string` — to gate auto-refresh interval
- `onFileSelect: (absolutePath: string) => void` — passed up to Phase 21 editor (no-op in Phase 20)
