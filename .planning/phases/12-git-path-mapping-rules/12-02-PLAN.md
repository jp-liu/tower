    State (per Research Pattern 7):
    ```typescript
    type RuleEditState = {
      host: string;
      ownerMatch: string;
      localPathTemplate: string;
      priority: number;
    };

    const [rules, setRules] = useState<GitPathRule[]>([]);
    const [editingId, setEditingId] = useState<string | null>(null);
    const [editForm, setEditForm] = useState<RuleEditState>({ host: "", ownerMatch: "*", localPathTemplate: "", priority: 0 });
    const [showAddForm, setShowAddForm] = useState(false);
    const [addForm, setAddForm] = useState<RuleEditState>({ host: "", ownerMatch: "*", localPathTemplate: "", priority: 0 });
    const [deleteConfirmId, setDeleteConfirmId] = useState<string | null>(null);
    ```

    Load on mount:
    ```typescript
    useEffect(() => {
      getConfigValue<GitPathRule[]>("git.pathMappingRules", []).then(setRules);
    }, []);
    ```

    Handlers (all use immutable patterns -- spread, filter, map -- NEVER mutate):

    - `handleAddRule`: **MUST validate before saving** -- verify `addForm.host.trim()` and `addForm.localPathTemplate.trim()` are non-empty. If either is empty, return early (no save). Creates new rule with `crypto.randomUUID()` for id, spreads into `[...rules, newRule]`, calls `setConfigValue("git.pathMappingRules", updated)`, updates local state, resets form, hides add form.

    - `handleEditStart(rule)`: Sets `editingId` to rule.id, populates `editForm` from rule fields.

    - `handleEditSave(ruleId)`: **MUST validate before saving** -- verify `editForm.host.trim()` and `editForm.localPathTemplate.trim()` are non-empty. If either is empty, return early (no save). Maps rules to produce updated array (`rules.map(r => r.id === ruleId ? { ...r, ...editForm } : r)`), calls `setConfigValue`, updates local state, clears editingId.

    - `handleEditCancel`: Sets editingId to null.

    - `handleDeleteRule(ruleId)`: Filters rules (`rules.filter(r => r.id !== ruleId)`), calls `setConfigValue`, updates local state, clears deleteConfirmId.
