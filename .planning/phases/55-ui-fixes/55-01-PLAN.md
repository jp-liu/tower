**UI-01 — Stop delete propagation (per D: stopPropagation on delete click handler):**

In `src/components/board/task-card.tsx`, the DropdownMenuItem for delete (line ~77-80) calls `onDelete?.(task.id)` directly. The DropdownMenuContent click event bubbles up to the card's `onClick`. Fix by wrapping the delete handler:

```tsx
<DropdownMenuItem
  className="text-rose-400"
  onClick={(e) => {
    e.stopPropagation();
    onDelete?.(task.id);
  }}
>
```

Also apply the same pattern to the edit DropdownMenuItem for safety:
```tsx
<DropdownMenuItem onClick={(e) => { e.stopPropagation(); onEdit?.(task); }}>
```

**UI-02 — Create shared EmptyState component:**

Create `src/components/ui/empty-state.tsx`:
```tsx
"use client";

import type { LucideIcon } from "lucide-react";
import type { ReactNode } from "react";

interface EmptyStateProps {
  icon?: LucideIcon;
  title: string;
  description?: string;
  action?: ReactNode;
}

export function EmptyState({ icon: Icon, title, description, action }: EmptyStateProps) {
  return (
    <div className="flex h-48 flex-col items-center justify-center gap-2 text-center">
      {Icon && <Icon className="h-10 w-10 text-muted-foreground/40" />}
      <p className="text-sm font-medium text-muted-foreground">{title}</p>
      {description && (
        <p className="text-xs text-muted-foreground/60">{description}</p>
      )}
      {action && <div className="mt-2">{action}</div>}
    </div>
  );
}
```
