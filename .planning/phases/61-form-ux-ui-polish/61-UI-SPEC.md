## Interaction Contracts

### FORM-01: Create Project — Plain Text localPath

- **Before:** `FolderBrowserDialog` browse button present, localPath auto-resolved and not user-editable
- **After:** Remove browse button. localPath renders as plain `<Input>` (editable). Auto-resolve from gitUrl continues to populate it via `resolveGitLocalPath`, but user can override by typing.
- **State:** When gitUrl changes and auto-resolve runs, localPath input value updates reactively. If user has manually typed a value, do NOT overwrite it (use `localPathManual` flag pattern already in `create-project-dialog.tsx`).
- **Visual:** Input height `h-8`, width `w-full`, same as all other fields. No lock icon.

### FORM-02: Import Project — Read-Only localPath After Browse

- **Behavior:** After user selects a folder via browse button, localPath Input becomes read-only.
- **Visual:** Add `readOnly` prop to Input + add `text-muted-foreground bg-muted/50` to signal non-editable state. Do NOT use `disabled` (disables form submission value).
- **Browse button:** Retain at 100% — no change.

### FORM-03: Migration — Editable Target Path

- **Behavior:** Target path field must be an editable `<Input>`. Verify it is not using `disabled` or `readOnly` in current implementation.
- **Visual:** Standard Input styling, no visual distinction needed.

### FORM-04: Textarea Max-Height

- **Apply to all three dialogs:** `CreateProjectDialog` description field, `ImportProjectDialog` description field, `CreateTaskDialog` description field.
- **CSS:** Add `max-h-[200px] overflow-y-auto` directly to the `<Textarea>` component usage (not a wrapping div).
- **Note:** The Textarea component uses `field-sizing-content` by default (auto-grows with content). Adding `max-h-[200px] overflow-y-auto` caps the growth. The scrollbar renders using the project's custom scrollbar style (6px width, transparent track).
- **Dialog height:** No dialog should grow to push action buttons off-screen. With 200px cap, dialogs remain within viewport.

### FORM-05: Clone Directory ~ Validation

- **Trigger:** When the clone directory settings input value starts with `~`.
- **Frontend:** Display warning label immediately below the input: text `"请输入绝对路径，不支持 ~ 别名"`, styled `text-xs text-destructive mt-1`.
- **Backend:** `resolveGitLocalPath` in `src/actions/config-actions.ts` must return a validation error for paths starting with `~`. Surface this error in the UI via `toast.error()` (Sonner).
- **Submit behavior:** Prevent form submission when the value starts with `~` (disable submit button or run client-side validation in `handleSubmit`).

### UI-01: Assistant Icon Relocation

- **Current position:** In the right actions area, between the search button and language toggle.
- **Required position:** Reading the top-bar source, the Bot icon is ALREADY placed immediately after the search button and before the language toggle. The CONTEXT.md decision to move it "to the right of the search box" is already satisfied in the current implementation.
- **Action:** Verify this is correct. If any future change places it elsewhere, the correct slot is: `search button → [gap 8px] → Bot icon button → language toggle`.
- **Icon button style (unchanged):** `rounded-lg p-2 text-muted-foreground transition-colors hover:bg-accent hover:text-foreground` with active state `bg-accent text-foreground` when `assistantOpen` is true.
- **Tooltip:** Add `<Tooltip>` with content `t("assistant.iconLabel")` on hover — improves discoverability. Use shadcn Tooltip component, `delayDuration={500}`.
