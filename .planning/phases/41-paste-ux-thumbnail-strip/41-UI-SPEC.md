## Component Inventory

### New: `useImageUpload` custom hook

File: `src/hooks/use-image-upload.ts`

State shape per pending image:
```typescript
interface PendingImage {
  id: string;          // crypto.randomUUID()
  file: File;
  blobUrl: string;     // URL.createObjectURL(file) — revoke on remove/unmount
  status: "uploading" | "done" | "error";
  progress: number;    // 0–100
  filename?: string;   // returned by Phase 40 API on success
}
```

Responsibilities:
- `addImages(files: File[])` — create blob URLs, initiate XHR uploads
- `removeImage(id: string)` — abort XHR, revoke blob URL, filter state
- `clearAll()` — revoke all blob URLs, reset state (called after send)
- XHR progress via `xhr.upload.onprogress` → updates `progress` field
- Cleanup: revoke all blob URLs on unmount via `useEffect` return

### New: `ImageThumbnailStrip` component

File: `src/components/assistant/image-thumbnail-strip.tsx`

Layout: horizontal `flex flex-row gap-2 px-4 pt-2 pb-0` row
Only rendered when `pendingImages.length > 0` (no layout shift when empty)

Each thumbnail:
```
<div className="relative h-12 w-12 shrink-0 rounded-md overflow-hidden ring-1 ring-border">
  <img src={blobUrl} className="h-full w-full object-cover" alt="" aria-hidden="true" />

  {/* Progress overlay — visible while status === "uploading" */}
  <div className="absolute inset-0 flex flex-col justify-end bg-black/30">
    {/* Bottom progress bar */}
    <div className="h-1 bg-primary transition-all duration-150"
         style={{ width: `${progress}%` }} />
    {/* Percentage text — centered */}
    <span className="absolute inset-0 flex items-center justify-center
                     text-[10px] font-semibold text-white">
      {progress}%
    </span>
  </div>

  {/* X remove button — always visible */}
  <button
    className="absolute top-0.5 right-0.5 h-4 w-4 rounded-full
               bg-black/60 text-white flex items-center justify-center
               hover:bg-black/80 transition-colors"
    aria-label={t("assistant.removeImage")}
    onClick={() => removeImage(id)}
  >
    <X className="h-2.5 w-2.5" />
  </button>
</div>
```

Error state (status === "error"): replace `ring-border` with `ring-destructive`; hide progress overlay; show error icon overlay (`AlertCircle h-4 w-4 text-destructive-foreground`).

Completion animation: progress bar width transitions to 100% then fades out (`opacity-0 transition-opacity duration-300 delay-200`) when status changes to "done".

## Interaction States

### Paste event flow

1. User pastes with Ctrl+V / Cmd+V while textarea is focused
2. `onPaste` fires — `e.clipboardData.items` iterated
3. For each item where `type.startsWith("image/")`:
   - `item.getAsFile()` → `File` object
   - `addImages([file])` → creates blob URL, sets status "uploading", starts XHR
4. Text items pass through to default textarea paste behavior unaffected
5. Thumbnail strip appears immediately with the new thumbnail at 0% progress

### Upload progress states

| State | Visual |
|-------|--------|
| uploading (0–99%) | Progress bar at bottom, percentage overlay, X button visible |
| uploading (100%) | Bar full, animating to done |
| done | Progress overlay fades out, image shows cleanly, ring-border |
| error | ring-destructive, AlertCircle icon overlay, no progress bar |

### Send disabled conditions

`isSendDisabled` is true when:
- Input text is empty AND no `done` images pending
- OR `isThinking` is true
- OR any image has `status === "uploading"` (wait for all uploads to complete)

### Thumbnail removal

- Click X button on any thumbnail regardless of upload status
- If status "uploading": abort XHR, revoke blob URL, remove from state
- If status "done"/"error": revoke blob URL, remove from state
- Strip disappears when `pendingImages.length === 0`

### After send

1. `handleSend` collects `filename` values from `done` images
2. Passes filenames to `sendMessage(text, { imageFilenames: string[] })`
3. `clearAll()` called — all blob URLs revoked, state reset to `[]`
4. Strip disappears, textarea resets

## Copywriting Contract

| Element | zh copy | en copy | i18n key |
|---------|---------|---------|----------|
| Primary CTA (send with images) | 发送 | Send | `assistant.sendLabel` (existing) |
| Remove image button aria-label | 移除图片 | Remove image | `assistant.removeImage` |
| Upload error thumbnail aria | 上传失败 | Upload failed | `assistant.uploadFailed` |
| Preview modal close aria-label | 关闭预览 | Close preview | `assistant.closePreview` |
| Zoom in cursor hint (alt) | (no copy — CSS cursor only) | (no copy — CSS cursor only) | — |
| Paste hint placeholder (optional) | 输入消息或粘贴图片... | Type a message or paste an image... | `assistant.inputPlaceholderWithImages` |
| Input placeholder (existing, unchanged) | 输入消息... | Type a message... | `assistant.inputPlaceholder` (existing) |

Note on placeholder: The "paste image" hint in the placeholder is listed under Claude's Discretion in CONTEXT.md. Decision: add a distinct i18n key `assistant.inputPlaceholderWithImages` but only display it when the thumbnail strip is empty. When images are pending, keep the existing short placeholder to reduce visual noise.

Empty state: Phase 41 adds no new empty state UI — the thumbnail strip simply does not render when `pendingImages.length === 0`.

Error state copy: Upload errors show `ring-destructive` on the thumbnail with an icon only — no text banner. Rationale: thumbnail strip is compact (48px); text error message would require layout expansion. The X button remains available to dismiss and retry by pasting again.

Destructive actions in this phase:
- "Remove image" (X button on thumbnail) — no confirmation dialog required. The action is immediately reversible by pasting again. Inline single-click removal follows established patterns for draft attachments.

Source: CONTEXT.md decisions; existing i18n patterns in `i18n.tsx`

## Accessibility Contract

- Thumbnail `<img>` uses `alt=""` and `aria-hidden="true"` — decorative, not content (user sees it, screen reader skips)
- Remove X button has `aria-label={t("assistant.removeImage")}` with image index context
- Preview modal is a `Dialog` — receives focus on open, returns focus on close, Escape closes it
- Progress bar uses `role="progressbar" aria-valuenow={progress} aria-valuemin={0} aria-valuemax={100}`
- Textarea `onPaste` does not call `e.preventDefault()` on text items — only suppresses default for image items
- Send button disabled state communicated via `disabled` attribute (not just visual)
