## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, bubble badge margins |
| sm | 8px | Bubble inner horizontal padding, icon-to-label gap |
| md | 16px | Chat panel body padding (horizontal), input area padding |
| lg | 24px | Section padding, send button gap from input |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Title bar height: 44px — inherits from Phase 37 `assistant-panel.tsx`
- Chat bubble vertical padding: 8px (`py-2`) — compact reading density matching sm token
- User bubble horizontal padding: 12px (`px-3`) — tighter than md to preserve bubble width budget
- Tool-call block vertical padding: 4px (`py-1`) — compact to keep tool calls visually subordinate
- Input textarea min-height: 40px — matches `h-10` for single-line state
- Send button: 32px (`h-8 w-8`) — matches existing button convention per `.claude/rules/ui.md`

Source: `.claude/rules/ui.md` button sizing, Phase 37 title bar dimensions

## Layout & Interaction Contract

### Chat Panel Structure

The `AssistantPanel` receives a new `renderMode` prop: `"terminal"` (existing, Phase 37) or `"chat"` (new, Phase 38). The `AssistantProvider` reads `assistant.displayMode` config key and passes it as `renderMode`.

```
AssistantPanel
├── Title bar (44px, unchanged from Phase 37)
├── Message list area (flex-1, ScrollArea)
│   ├── [assistant bubble] [thinking bubble] [tool-call block] ...
│   └── [user bubble]
└── Input area (auto-height, max 120px)
    ├── Textarea (flex-1, Enter=send, Shift+Enter=newline)
    └── Send button (h-8 w-8, ghost variant, ArrowUp icon)
```

### Bubble Layout

| Message Role | Alignment | Max Width | Border Radius |
|---|---|---|---|
| `user` | Right-aligned (`justify-end`) | `max-w-[80%]` | `rounded-2xl rounded-br-sm` |
| `assistant` | Left-aligned (`justify-start`) | `max-w-[85%]` | `rounded-2xl rounded-bl-sm` |
| `thinking` | Left-aligned (`justify-start`) | `max-w-[200px]` | `rounded-2xl rounded-bl-sm` |
| `tool` | Left-aligned (`justify-start`) | `max-w-[90%]` | `rounded-lg` |

Message list: `flex flex-col gap-3 p-4` on the inner scroll container. Auto-scroll to bottom on new messages via `useEffect` on messages array length.

### Thinking Indicator

Three pulsing dots, each `size-1.5 rounded-full bg-primary/60`, staggered `animate-pulse` with `animation-delay` of 0ms, 150ms, 300ms. Contained in a `bg-muted/50` bubble, left-aligned. Visible only when `isThinking === true`. Hidden immediately when assistant response segment starts.

### Tool-Call Block

Collapsed by default. Shows: `ChevronRight` icon (rotates to `ChevronDown` when expanded) + tool name text at 12px semibold + "Tool" badge (`bg-muted text-muted-foreground text-[10px]`). Expanding reveals the tool result content in a `font-mono text-[12px]` pre-scrollable block capped at `max-h-[200px] overflow-y-auto`.

### Input Area

Height: auto-resizing, single line minimum (40px), maximum 120px before scrolling. `Shift+Enter` inserts a `\n`. `Enter` (without Shift) calls `onSend()` — forwards message text + `\n` to PTY via WebSocket, then clears input. Auto-focus on panel open: `useEffect` with `inputRef.current?.focus()` when `isOpen === true`.

Send button: disabled when input is empty or whitespace-only, or when `isThinking === true`. Uses `Button variant="ghost" size="icon"` with `SendHorizonal` icon (16px) from lucide-react.

### Empty State (no messages yet)

Centered in the message list area. Shows `Bot` icon (32px, `text-muted-foreground/40`) + heading + body text. No action button — user types to start.

### Scroll Behavior

`ScrollArea` wraps the message list. On new message appended: `scrollAreaRef.current?.scrollTo({ top: scrollAreaRef.current.scrollHeight, behavior: "smooth" })`.

### WebSocket Data Interception

In `"chat"` renderMode, `useAssistantChat` hooks into the same WebSocket connection from `AssistantProvider`. Raw WebSocket `message` events are intercepted — data is NOT written to xterm.js (that path is skipped). Instead, the raw string chunk is passed to the incremental state-machine parser. The terminal xterm instance is not created in chat mode — `DynamicTerminal` is not mounted.

Source: `38-CONTEXT.md` integration decisions

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Send on Enter | `onKeyDown`: `if (e.key === "Enter" && !e.shiftKey) { e.preventDefault(); onSend(); }` |
| Input accessible label | `aria-label={t("assistant.chat.inputLabel")}` on Textarea |
| Message list role | `role="log" aria-live="polite" aria-label={t("assistant.chat.messageList")}` on message container |
| User bubble identification | `aria-label="You: {message content truncated to 60 chars}"` |
| Assistant bubble identification | `aria-label="Assistant: {message content truncated to 60 chars}"` |
| Thinking indicator | `aria-live="polite"` announces when thinking starts/stops; `role="status"` |
| Focus management | Input auto-focuses on panel open; focus returns to input after send |
| Tool-call toggle | `aria-expanded`, `aria-controls` on the toggle button |
