<objective>
Create two internal HTTP routes that bridge external processes (MCP stdio) to in-memory PTY sessions.

Purpose: The MCP stdio process runs in a separate Node.js process with its own globalThis, so it cannot import the PTY session registry directly. These HTTP routes on localhost:3000 act as the IPC channel: MCP tools (Phase 34) will call these routes to read terminal output and send input.

Output: Two App Router route handlers at `/api/internal/terminal/[taskId]/buffer` (GET) and `/api/internal/terminal/[taskId]/input` (POST), plus a shared localhost guard utility.
</objective>

  <action>
Create `src/lib/internal-api-guard.ts` with a single exported function:

```typescript
export function requireLocalhost(request: NextRequest): NextResponse | null
```

Logic:
1. Extract the requesting host from `request.headers.get("host")` and check it starts with `localhost` or `127.0.0.1` or `[::1]`.
2. Also check `request.headers.get("x-forwarded-for")` — if present, all IPs must be loopback (`127.0.0.1`, `::1`).
3. If the check fails, return `NextResponse.json({ error: "Forbidden" }, { status: 403 })`.
4. If the check passes, return `null` (meaning "proceed").

Import `NextRequest, NextResponse` from `next/server`.

This utility will be called at the top of every internal route handler:
```typescript
const blocked = requireLocalhost(request);
if (blocked) return blocked;
```

IMPORTANT: Do NOT use `request.ip` — it is not reliably populated in all Next.js runtimes. Use header-based detection only.

Create the App Router route handler at `src/app/api/internal/terminal/[taskId]/buffer/route.ts`.

The GET handler:

1. Call `requireLocalhost(request)` — return 403 if non-null.
2. Extract `taskId` from route params: `const { taskId } = await params` (Next.js 15 async params pattern — params is a Promise in App Router route handlers).
3. Call `getSession(taskId)` from `@/lib/pty/session-store`.
4. If no session found, return `NextResponse.json({ error: "No active session" }, { status: 404 })`.
5. Call `session.getBuffer()` to get the raw buffer string.
6. Split buffer by newlines, take the last N lines (N from query param `?lines=100`, default 100, max 500).
7. Return `NextResponse.json({ taskId, lines: linesArray, total: linesArray.length, killed: session.killed })`.

Use Zod to validate the `lines` query parameter: `z.coerce.number().int().min(1).max(500).default(100)`.

CRITICAL: This handler MUST be able to see the same `globalThis.__ptySessions` Map that instrumentation.ts populates. This works because:
- `session-store.ts` uses `globalThis.__ptySessions` (not module-level variable)
- `instrumentation.ts` imports `ws-server.ts` which imports `session-store.ts`, initializing the Map
- App Router route handlers run in the same Node.js process

Add `export const dynamic = "force-dynamic"` to prevent caching.
Add `export const runtime = "nodejs"` to ensure Node.js runtime (not Edge).
