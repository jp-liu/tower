Modify the PtySession class in src/lib/pty/pty-session.ts:

1. **envOverrides parameter** (NTFY-01):
   - Add `envOverrides?: Record<string, string>` as a new constructor parameter after `onExit`.
   - In the `pty.spawn()` call, spread `...envOverrides` AFTER the existing env object keys so overrides take precedence. NEVER mutate `process.env` — the override is local to the spawn call only.
   - The existing env keys (PATH, HOME, SHELL, TERM, LANG, USER, ANTHROPIC_API_KEY) remain unchanged.

2. **Idle detection** (NTFY-06, NTFY-07):
   - Add private fields:
     ```
     private _idleTimer: ReturnType<typeof setTimeout> | null = null;
     private _idleThresholdMs: number;
     private _onIdle: (() => void) | null;
     private _idleFired = false;
     ```
   - Add constructor parameters: `onIdle?: () => void` and `idleThresholdMs?: number` (default `180_000`, minimum 180s per STATE.md decision).
   - Store `_onIdle = onIdle ?? null` and `_idleThresholdMs = Math.max(idleThresholdMs ?? 180_000, 180_000)`.
   - Create a private method `_resetIdleTimer()`:
     - Clear existing `_idleTimer` if set.
     - If `_onIdle` is null or `killed` is true or `_idleFired` is true, return.
     - Set `_idleTimer = setTimeout(() => { this._idleFired = true; this._onIdle?.(); }, this._idleThresholdMs)`.
   - Call `_resetIdleTimer()` inside the `_pty.onData` callback (after buffer update, before calling `_onData`). This ensures every PTY output resets the timer.
   - In the `write()` method, call `this._resetIdleTimer()` BEFORE `this._pty.write(data)` — user input resets idle timer (NTFY-07).
   - In the `_pty.onExit` callback, clear the idle timer: `if (this._idleTimer) { clearTimeout(this._idleTimer); this._idleTimer = null; }`.
   - Call `_resetIdleTimer()` once at the end of the constructor to start the initial idle countdown.

3. **Public method for idle state**:
   - Add `get isIdle(): boolean { return this._idleFired; }` — Phase 34 MCP tools will use this.
