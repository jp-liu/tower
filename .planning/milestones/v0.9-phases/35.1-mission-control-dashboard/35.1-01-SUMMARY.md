## What Was Built

**Server Action (`getActiveExecutionsAcrossWorkspaces`):**
- Queries `TaskExecution` records with `status: "RUNNING"` using full Prisma join chain: execution → task → project → workspace
- Returns `ActiveExecutionInfo[]` with all fields needed to render mission cards
- `startedAt` serialized as ISO string for safe server-to-client serialization

**Grid Layout Presets (`GRID_PRESETS`):**
- 6 presets: 1×1, 2×1, 2×2, 3×2, 4×2, 3×3
- Each preset carries `cols`, `rows`, `minHeight` for CSS grid styling
- `DEFAULT_PRESET_ID = "2x2"`

**`/missions` Route:**
- Server component (`page.tsx`) fetches initial executions and passes to `MissionsClient`
- `MissionsClient` holds `cards` state, `presetId` state (localStorage-backed), `launcherOpen` state
- Toolbar with grid preset Select (using manual `<span>` per ui.md, not SelectValue) and Launch Task button
- Empty state: Rocket icon + headings + CTA button when no running executions
- Card grid: CSS grid with `gridTemplateColumns` and `gridAutoRows` derived from selected preset
- Placeholder cards showing task title for non-empty state

**Sidebar Navigation:**
- Collapsed view: Gauge icon tooltip button with active state styling, placed before Settings
- Expanded view: Footer Link with Gauge icon, unconditionally visible above workspace-specific links
- Both views use `pathname === "/missions"` for active state

**i18n Keys:**
- 23 keys added in both `zh` and `en`: `missions.navLabel`, `missions.pageTitle`, `missions.launchTask`, `missions.emptyTitle`, `missions.emptyDesc`, `missions.gridPreset`, `missions.stopTask`, `missions.openFullView`, `missions.runningTime`, `missions.statusCompleted`, `missions.statusStopped`, and all `missions.launcher.*` and `missions.error.*` keys

**Tests:**
- `agent-actions.test.ts` extended with 3 tests for `getActiveExecutionsAcrossWorkspaces` (empty array, RUNNING executions with join chain fields, excludes COMPLETED/FAILED)
- `missions-merge.test.ts` created with 5 pure function tests for poll merge logic

## Decisions Made

- `ActiveExecutionInfo.startedAt` serialized as ISO string — Date objects cannot cross server/client boundary in Next.js Server Components
- Missions sidebar link is unconditionally visible (not inside `activeWorkspaceId` conditional) — Missions is a global view, not workspace-scoped
- Grid preset Select uses `<span>` to display label (not `<SelectValue>`) — per project ui.md convention
- `mergeMissions` function defined inline in test file — will be extracted to util in Plan 02

## Deviations from Plan

None — plan executed exactly as written.

## Known Stubs
