**Accent (`--primary`) reserved for:**
- "Launch Task" primary CTA button (filled)
- Active/selected state of the sidebar `/missions` nav item (text color)
- Ring/focus outline on focused interactive elements (via `--ring`)
- Status badge: RUNNING badge border/text accent

**Accent must NOT be used for:**
- Card backgrounds
- Breadcrumb text
- Grid container
- Stop button (use destructive)
- Completed/Stopped status badges (use muted)

**Terminal background:** `#0a0a0a` — xterm.js internal background, near-black, always dark regardless of app theme. This is handled by `TaskTerminal` internally and matches the existing pattern.

**Status badge colors:**
- RUNNING: `text-primary border border-primary/30 bg-primary/10` (amber accent)
- Completed (fade-out): `text-muted-foreground bg-muted` (neutral)
- Stopped (fade-out): `text-muted-foreground bg-muted` (neutral)

## Layout Contract

### Page Layout (`/missions`)

The primary visual anchor of the `/missions` screen is the MissionCard grid: a uniform tiled layout where each card occupies equal space and the embedded terminal output draws the eye immediately, making the live execution state the dominant focal point.

```
┌─────────────────────────────────────────────────────────┐
│ Sidebar (existing app-sidebar.tsx, unchanged width)      │
│  [+ /missions nav item in footer — always visible]       │
├─────────────────────────────────────────────────────────┤
│ Toolbar (h-12, px-4, border-b border-border)             │
│  Left: page title (text-base font-semibold)              │
│  Right: grid preset Select (w-32 h-8) + Launch Task Btn  │
├─────────────────────────────────────────────────────────┤
│ Grid area (flex-1, overflow-auto, p-4)                   │
│  CSS Grid: grid-template-columns: repeat(N, 1fr)         │
│  gap-4 (16px) between cards                              │
│  Each cell: min-h-[240px]                                │
│                                                          │
│  Empty state: h-full flex flex-col items-center          │
│               justify-center gap-4                        │
└─────────────────────────────────────────────────────────┘
```

### Grid Presets

| Preset ID | Label | Columns | Rows | Min card height |
|-----------|-------|---------|------|-----------------|
| 1x1 | 1×1 | 1 | 1 | 480px |
| 2x1 | 2×1 | 2 | 1 | 480px |
| 2x2 | 2×2 | 2 | 2 | 300px |
| 3x2 | 3×2 | 3 | 2 | 300px |
| 4x2 | 4×2 | 4 | 2 | 240px |
| 3x3 | 3×3 | 3 | 3 | 240px |

CSS implementation: `grid-template-columns: repeat({cols}, 1fr)`. Height is auto from content; cards stretch to fill grid row height via `grid-auto-rows: minmax({minHeight}, 1fr)`.

Default preset: `2x2` (used when no localStorage value set).

### MissionCard Layout

```
┌──────────────────────────────────────────────┐
│ Card header (h-10, px-3, flex items-center)   │
│  ≡  Workspace › Project › Task Title         │
│  [RUNNING] Xm Ys  [↗] [×]                   │
├──────────────────────────────────────────────┤
│ Terminal area (flex-1, overflow-hidden)        │
│  TaskTerminal (h-full w-full)                 │
│  Loading skeleton: bg-[#0a0a0a] animate-pulse │
└──────────────────────────────────────────────┘
```

Card header breakdown (left to right):
1. Drag handle `≡` — `cursor-grab text-muted-foreground hover:text-foreground` — `GripVertical` icon, 14px, mr-2
2. Breadcrumb — `text-[11px] text-muted-foreground truncate flex-1` — "WS › Project › Task" (workspace and project in muted, task title in `text-foreground font-semibold`)
3. Status badge — `text-[11px]` — RUNNING badge (amber), or Completed/Stopped (muted) during fade-out — ml-2
4. Running time — `text-[11px] text-muted-foreground ml-1` — "Xm Ys"
5. Open full view button — `Button` default size, ghost variant, icon-only (`ArrowUpRight` 14px) — ml-1
6. Stop button — `Button` default size, ghost variant, icon-only (`X` 14px, `text-destructive hover:bg-destructive/10`) — ml-1

Card fade-out: `transition-opacity duration-300` — opacity goes from 1 to 0 over 300ms when card enters `removingIds` state, then removed from DOM after 3000ms total.

Card border: `border border-border rounded-lg overflow-hidden` — inherits radius from `--radius` (0.5rem).

## Interaction Contract

### Grid Preset Select (Toolbar)

- Component: shadcn `Select` with `SelectTrigger` (h-8, w-32)
- Label display: Manual `<span>` showing selected preset label — NEVER use `<SelectValue />`
- Example: `<span className="truncate">{GRID_PRESETS.find(p => p.id === preset)?.label ?? preset}</span>`
- On change: update React state + write to `localStorage("missions-grid-preset", newId)`
- On mount: read `localStorage("missions-grid-preset") ?? "2x2"` in useState initializer (with `typeof window !== "undefined"` guard)

### Launch Task Dialog (Task Picker)

- Trigger: "Launch Task" Button in toolbar (primary variant, default size h-8)
- Dialog width: `max-w-md`
- Content: Three stacked Selects
  1. Workspace select — loads from `getWorkspaces()` on dialog open
  2. Project select — loads from workspace's projects when workspace selected; disabled until workspace selected
  3. Task select — loads from `getProjectTasks(projectId)`, filtered to TODO + IN_PROGRESS; disabled until project selected; empty state text: `t("missions.launcher.noTasks")`
- All Select triggers: manual `<span>` pattern (see Select rule)
- Footer: "Cancel" (ghost) + "Launch Task" (primary, disabled until task selected)
- On Launch Task click: call `startPtyExecution(taskId)`, close dialog, add new card to grid optimistically

### Stop Button (per card)

- Icon: `X` from lucide-react, 14px
- Tooltip: `t("missions.stopTask")`
- Click: call `stopPtyExecution(executionId)`, immediately add card to `removingIds` (do not remove from `cards` yet — poll merge will handle deduplication)
- No confirmation dialog — stop is immediate

### Open Full View Button (per card)

- Icon: `ArrowUpRight` from lucide-react, 14px
- Tooltip: `t("missions.openFullView")`
- Click: `router.push("/workspaces/{workspaceId}/tasks/{taskId}")`

### Drag-to-Reorder

- Library: `@dnd-kit/sortable` with `rectSortingStrategy`
- `DndContext` must have `id="missions-dnd"` (stable, prevents hydration mismatch)
- `activationConstraint: { distance: 8 }` (matches kanban-board.tsx sensor config)
- Drag handle: `≡` (`GripVertical` icon) — activates drag on the card
- Order is local state only — no server persistence

### Polling & Auto-Remove

- Interval: 4000ms (within the 3-5 second window from D-10; Claude's discretion)
- Poll function: `getActiveExecutionsAcrossWorkspaces()` server action
- Card appears: new execution ID in poll result → append to cards array
- Card disappears: execution ID absent from poll result → add to `removingIds`, remove from cards after 3000ms
- Poll must be cleared in `useEffect` cleanup — `clearInterval(timer)`
- All fade `setTimeout` handles must be tracked in `useRef<Map<string, ReturnType<typeof setTimeout>>>` and cleared on unmount

### Terminal Theme Matching

- Follow existing `TaskTerminal` pattern: reads `resolvedTheme` from `next-themes`
- Light theme terminal: background `#1e1e1e`, foreground `#d4d4d4` (VS Code light-dark convention already used in TaskTerminal)
- Dark theme terminal: background `#0a0a0a`, foreground `#f0f0f0`
- No new theme logic needed — reuse `TaskTerminal` unchanged

### Loading State (per card terminal area)

- While `TaskTerminal` dynamic import resolves: `<div className="h-full w-full bg-[#0a0a0a] animate-pulse" />`
- While WebSocket connecting: `TaskTerminal` shows its own `wsStatus: "connecting"` state (handled internally)

### Sidebar Nav Item

- Route: `/missions`
- Icon: `Gauge` from lucide-react (monitoring/control semantic)
- Active state: `pathname === "/missions"` → `text-foreground` (not primary/amber — matches existing sidebar footer link active pattern)
- Inactive state: `text-muted-foreground hover:text-foreground`
- Placement: In sidebar footer section, unconditionally visible (before workspace-scoped conditional blocks)
- Collapsed view: `Tooltip`-wrapped icon button at 32px (`h-8 w-8`), tooltip text: `t("missions.navLabel")`
- Expanded view: icon + label in same row pattern as existing footer links

**Destructive actions in this phase:**
- Stop task execution — triggered via Stop button (×) on card
- Confirmation approach: NO confirmation dialog — stop is immediate, card enters 3-second fade state
- Rationale: Fast operator action; fade-out provides visual feedback; user can navigate to task detail for more context

**Error states:**

| Scenario | Toast message |
|----------|---------------|
| `startPtyExecution` fails | `toast.error(t("error.startExecutionFailed"))` — use existing error key if available, else add new key "missions.error.launchFailed" |
| `stopPtyExecution` fails | `toast.error(t("missions.error.stopFailed"))` |
| Server action fetch fails during poll | Silent — do not show toast on poll failure; retry on next tick |
