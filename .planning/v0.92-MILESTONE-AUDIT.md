## Integration Issues

### UI-05 Config Propagation (Minor)

`AssistantProvider` reads `assistant.communicationMode` in a `useEffect` with empty dependency array — runs once on mount. When a user changes the mode in Settings > General, the DB is updated but the live React context is not refreshed. The panel uses the mode from initial mount until a page reload occurs.

**Impact:** Users must reload the page after changing communication mode for the setting to take effect. Functionality is correct after reload.

**Fix:** Add a re-read mechanism (e.g., read config when panel opens, not just on mount).
